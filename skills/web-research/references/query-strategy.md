# Query Strategy

Use this reference when the research task needs a better search plan.

## Goal Framing

- Restate the exact fact to verify before searching.
- Extract unstable elements first: date, version, price, release, office holder, policy, schedule.
- Separate the answer into sub-questions when one search would mix multiple facts.

## Query Patterns

- Current fact check:
  - `<topic> official`
  - `<topic> site:official-domain`
  - `<topic> latest`
- Technical documentation:
  - `<library> official docs <feature>`
  - `site:docs.<vendor-domain> <feature>`
  - `site:github.com <project> release notes <version>`
- News or recent events:
  - `<subject> news`
  - `<subject> announced <month> <year>`
  - `<subject> site:reuters.com OR site:apnews.com`
- Product or pricing:
  - `<product> pricing official`
  - `<product> plans official`
  - `<product> compare official`

## Narrowing Rules

- Add a domain filter when a primary source exists.
- Add a recency filter for unstable topics.
- Search by exact product, package, or company name before adding broad context words.
- Use multiple focused searches instead of one overloaded search.

## Verification Pass

After the first promising result:

1. Open the primary source.
2. Confirm the publication date or effective date.
3. Cross-check one independent source when the claim affects money, legal risk, health, or business decisions.
